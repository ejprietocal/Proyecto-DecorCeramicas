# Decorcerámica ERP — AGENTS.md

## Project structure

All code lives in `Proyecto-DecorCeramicas/`. This is a **no-build vanilla JS multi-page app** with an in-memory mock database backed by localStorage. Open any `.html` file directly in a browser — no server, no install.

Each module is its own HTML file with its own inline JS.
- `index.html` — login page (auth → stores user in **sessionStorage**, redirects to `dashboard.html`)
- `dashboard.html`, `pedidos.html`, `inventario.html`, `sobrantes.html`, `movimientos.html`, `transferencias.html`, `proveedores.html`, `compras.html`, `reclamar.html`, `vencimientos.html`, `usuarios.html`

## Language

Everything is in **Spanish**: UI text, code comments, variable names, commit messages.

## Shared JS — loading order (critical)

Each module page loads shared JS in this order (matching `index.html:120-129` original):

```
config.js → db.js → helpers.js → operations.js → [draft.js] → auth.js → ui.js → (inline page JS)
```

`draft.js` is only loaded by pages that need line-item modals (pedidos, compras, transferencias).

## Auth & navigation

- **Login**: `index.html` → users are **numbers 1–6** (1=Admin … 6=Auditora), password is arbitrary. See `config.js:16-23`.
- `currentUser` is stored in **sessionStorage** and restored on every page via `renderShell(moduleId)` in `auth.js`.
- Navigation uses `<a href="page.html">` links in the sidebar (generated by `buildSb()` in `auth.js`).
- `renderShell()` in `auth.js:53` checks auth, builds sidebar, sets topbar title/icon/breadcrumb, and renders user info. Every module page calls it first.

## Persistence

- **DB** is persisted to **localStorage** under key `decorceramica_db` (via `saveDB()` in `db.js:70`). Saved automatically on `beforeunload`.
- **Auth** is persisted to **sessionStorage** under key `decorceramica_user` (cleared on tab close or logout).
- `saveDB()` must be called explicitly after every mutation if the agent modifies data inline.

## Key globals

- `DB` — in-memory database (in `db.js`), loaded from localStorage on init
- `$()` — alias for `document.getElementById` (in `helpers.js:2`)
- `fmt()`, `money()`, `now()` — formatting helpers (in `helpers.js`)
- `currentUser`, `selBodega` — session state (in `db.js:66-67`)
- `PERMS` — per-user module access control (in `config.js:24-31`)
- `draft` — temporary line-item buffer for modal forms (in `draft.js`)
- `PAGE_MAP` — moduleId → filename lookup (in `auth.js:4-8`)
- `MOBS` — module definitions (icon, name, group, breadcrumb; in `config.js:2-14`)

## Obsolete files (do not use)

- `js/init.js` — no longer loaded (VIEWS object was SPA-only)
- `js/modals.js` — each page inlines its own modal functions
- `js/views/*.js` — view rendering is inline in each HTML page

## Business flow

1. **Pedido de Venta** → system checks stock across all warehouses → assigns optimal warehouse → status: `pendiente → asignado/resagado → despachado`
2. **OC (Orden de Compra)** → `transito → recibida → reclamada` (adds stock to BGA on claim)
3. **Transferencia** between warehouses → `solicitada → autorizada → despachada → recibida`
4. Re-assigning resagado pedidos happens automatically after stock-affecting operations (`reclamarOC`, `recibirTransferencia`, `confirmarAjuste`, `reintegrarSobrante`)

## Deployment (Easypanel)

A `Dockerfile` at repo root serves `Proyecto-DecorCeramicas/` with `nginx:alpine` on port 80. Point Easypanel to this repo, the Dockerfile is detected automatically. No build step needed — it is a static site.
